Welcome to CivicFix. Tell me what's happening in your area — a pothole, garbage pile-up, broken streetlight, or water issue — and I'll take care of the rest.
// THE PROBLEM

Civic complaints get lost between citizens and city hall

Reporting a broken streetlight shouldn't require navigating a government form built for a case worker, not a citizen.

CASE 00 — FRICTION

Forms built for staff

Rigid categories and lengthy fields make reporting an everyday issue unnecessarily hard for the person who noticed it.

CASE 00 — NOISE

The same pothole, five times

One real issue gets reported separately by five neighbours, fragmenting priority instead of raising it.

CASE 00 — SILENCE

No one follows up

Once filed, complaints vanish into inboxes with no tracking, no accountability, and no automatic follow-up.

11.2M+

Grievances redressed via India's centralized CPGRAMS system between 2019–2024 — yet most systems still prioritize administrative process over the person filing the report.

// CASE LIFECYCLE

What happens after you hit send

Every message runs through a real agentic loop — not a static form handler.

STAGE 01

Report

You describe the issue in plain language — text, no dropdowns, no subcategories to hunt through.

STAGE 02

Classify

The agent identifies category and urgency, and asks for a location only if you haven't given one.

STAGE 03

De-duplicate

It checks nearby open cases and merges matching reports instead of filing a fresh duplicate.

STAGE 04

Track & nudge

A separate agent monitors open cases and drafts a follow-up if nothing moves.

// STANDOUT FEATURE

It recognizes when an issue is already known

Instead of filing a duplicate, the agent reasons about similarity — same category, same block, same problem — and merges the report.
